AgriOT – Greenhouse IoT Nodes

ESP32 RS-485 nodes reading soil sensors (moisture / pH / NPK), with MQTT control, OTA firmware, and Grafana dashboards for growers.

ESP32 / ESP-IDF FreeRTOS RS-485 (Modbus) MQTT Flask / Node.js Grafana

Overview

The AgriOT project is about building a fully automated greenhouse monitoring and irrigation system with the goal of increasing crop yield in controlled environments. The vision is to have sensor nodes spread across the greenhouse — each covering a cluster of plants or crops — that continuously measure critical environmental and soil conditions.

Each ESP32-based RS-485 node reports values like ambient temperature, humidity, soil temperature, soil moisture, soil pH, and light intensity. Based on this data, the system automatically irrigates each crop or plant through per-zone pump/valve control. All readings and states are published over MQTT, stored in databases, and visualized in Grafana dashboards for real-time insight.

This setup not only ensures optimal irrigation but also builds a large dataset that can later be used to train machine learning models for crop growth prediction and yield estimation. The idea is to move towards predictive agriculture where future irrigation schedules and growth forecasts are guided by historical data.

Architecture
  • ESP32 firmware (C/ESP-IDF) with FreeRTOS tasks: sensors, uplink, ota, control.
  • RS-485 (Modbus-RTU) multi-drop bus for pH/moisture/NPK sensors; optional flow meter & pump relay.
  • MQTT broker (Mosquitto/EMQX). Topics: agriot/<nodeId>/tele, .../state, .../cmd, .../ota.
  • Backend bridge (Python/Node) for rules, acks, and persistence to PostgreSQL/Influx.
  • Grafana dashboards + alerting for thresholds and offline nodes.

Key FW Features

  • FreeRTOS-based real-time system with dedicated tasks
  • MQTT-based topic logging & subscription
  • MQTT-driven OTA update pipeline
  • Motor (pump/valve) control with feedback
  • Improved error handling & reconnection strategies

Hardware

  • MCU: ESP32-WROOM-32.
  • Comms: RS-485 transceiver, ESD protection, bias/termination jumpers.
  • Power: 12 V input → buck 5 V / 3.3 V rails, reverse/over-voltage protection.
  • I/O: Relay driver for pump/valve, flowmeter input, I²C header for expansion.
