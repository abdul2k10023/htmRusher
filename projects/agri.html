AgriOT – Greenhouse IoT Nodes

ESP32 RS-485 nodes reading soil sensors (moisture / pH / NPK), with MQTT control, OTA firmware, and Grafana dashboards for growers.

ESP32 / ESP-IDF FreeRTOS RS-485 (Modbus) MQTT Flask / Node.js Grafana
GitHub/agriFW

Overview

The idea for AgriOT came from my own background in Sindh, Pakistan, where my family has been farming for generations. Cultivating crops there is extremely challenging — water scarcity dominates most of the year, temperatures are harsh, and growing off-season vegetables is nearly impossible without precise control of soil and climate conditions. Agriculture in Sindh is mostly focused on traditional cash crops such as sugarcane, cotton, rice, and wheat, while modern approaches like greenhouses are rarely adopted. Yet, greenhouses could enable farmers to diversify into high-value off-season vegetables and fruits that currently only thrive in the northern regions.

Seeing these challenges firsthand, I wanted to explore how data and automation could transform farming. I began by experimenting with simple soil and temperature sensors on small coriander and basil plants, just to see if the data would be useful. Over time, this grew into a complete system: adding DC motors for irrigation, moving from bare-metal programming to a structured FreeRTOS design, and eventually integrating OTA updates for continuous development

Today, AgriOT is about building a fully automated greenhouse monitoring and irrigation system with the goal of increasing crop yield in controlled environments. Sensor nodes are spread across the greenhouse — each covering a cluster of plants or crops — continuously measuring critical environmental and soil conditions.

Working PCB sending data to PostgreSQL DB

Each ESP32-based RS-485 node reports values like ambient temperature, humidity, soil temperature, soil moisture, soil pH, and light intensity. Based on this data, the system automatically irrigates each crop or plant through per-zone pump/valve control. All readings and states are published over MQTT, stored in databases, and visualized in Grafana dashboards for real-time insight.

Grafana Dashboard

This setup not only ensures optimal irrigation but also builds a large dataset that can later be used to train machine learning models for crop growth prediction and yield estimation. The vision is to move towards predictive agriculture, where irrigation schedules and growth forecasts are guided by historical and sensor-driven data.

Key FW Features

  • FreeRTOS-based real-time system with dedicated tasks
    • SHT 41 Sensor Task
    • RS485 Soil Sensor Task
    • MQTT Uplink Task
    • OTA Task
    • Motor Control Task
  • MQTT-based topic logging & subscription
  • MQTT-driven OTA update pipeline
  • Motor (pump/valve) control with feedback
  • Improved error handling & reconnection strategies

Planned FW features

  • Saving all crucial runtime parameters in NVS
  • Implementing actual ACK/NAK responses to commands
  • MQTT over TLS for secure connectivity
  • Low-power modes (sleep / deep sleep) for energy efficiency
  • Storing PKI credentials securely in EEPROM
  • Buffered data backup in case of connection loss
  • Plug & Play RS485 sensor using load switches

Hardware

  • MCU: ESP32-WROOM-32.
  • Comms: 2x RS-485 dedicated circuitry, I²C, UART .
  • Power: 12 V input → buck 5 V / 3.3 V rails, reverse/over-voltage protection.
  • I/O: 2x Solenoid Valve Control Outputs, 1x 12vdc Motor Control, 2x RS485 Soil Sensor Inputs

Architecture
  • ESP32 firmware (C/ESP-IDF) with FreeRTOS tasks for sensors, uplink, OTA, and control.
  • RS-485 (Modbus-RTU) multi-drop bus for soil sensors (Temperature, moisture, pH); plus SHT41 for Ambient temp/humidity.
  • MQTT broker on Raspberry Pi (with TLS), bridging ESP32 nodes to the server.
  • Server stack: PostgreSQL (storage), PostgREST (API), Grafana (dashboards & alerts).
  • Motor and solenoid control for automated irrigation, with power management and OTA updates.
